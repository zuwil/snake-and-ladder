How to Use

! Page Layout
Left Panel — Board configuration & settings
Center Panel — Game board, dice & play log
Right Panel — Simulation, board editor & statistics
1 Configure Your Board

Use the left panel to adjust settings, then click "Apply Settings & Generate Board" to generate. A simulation runs automatically.

SettingHow it works
Board WidthNumber of columns (3–20). Combined with height to determine total tiles (width × height). E.g. 10×10 = 100 tiles, 5×5 = 25 tiles. Total tiles directly affects game length — the base rolls needed (with no snakes/ladders) is approximately total tiles ÷ 3.5 (since average dice roll = 3.5).
Board HeightNumber of rows (3–20). Works together with width. Larger boards allow more room for snakes, ladders, and rewards. The board is laid out in a zigzag pattern (left-to-right, then right-to-left on the next row).
Rewards QuantityHow many reward tiles (0–50) to place. Rewards are randomly distributed with a no-adjacency rule — no two rewards can be next to each other (horizontally, vertically, or diagonally). Rewards also cannot be placed on tiles already occupied by snakes or ladders.
Target Rewards Per RunThe desired median number of rewards collected per game. This is a soft target — the system places rewards in reachable positions, but the actual median depends on where snakes/ladders route the player. The simulation result shows the true median achieved.
Target Avg RollsThe desired median rolls to finish. The algorithm uses a math model to calculate how many snakes and ladders to place:
• Base rolls = total tiles ÷ 3.5
• Each snake adds ~0.49 × drop distance extra rolls
• Each ladder saves ~0.49 × climb distance rolls
• The algorithm fixes 1–3 ladders (based on board size), then solves for how many snakes + how far they drop to hit the target
• Max snakes/ladders capped at 15% of total tiles each
Simulation Sample SizeHow many complete games to simulate (1K–2M). Each game plays from tile 1 to the final tile, rolling dice, hitting snakes/ladders, and collecting rewards. Higher values give more statistically accurate results but take longer. Runs in a background Web Worker so the page stays responsive.
Limitations & What's Not Achievable
• Board too small (under 3×3 = 9 tiles) — no snakes, ladders, or rewards will be placed at all. The algorithm skips placement entirely.
• Target Avg Rolls lower than base rolls — on a 10×10 board, base rolls ≈ 29. Setting target below this means the algorithm needs lots of ladders, but it only places 1–3 ladders. If the target is extremely low (e.g. target 5 on a 100-tile board), the result will still be close to base rolls since there aren't enough ladders to bridge the gap. No snakes will be placed in this case.
• Target Avg Rolls extremely high — the algorithm tries to place more/longer snakes, but snakes and ladders are each capped at 15% of total tiles. On a 100-tile board, that's max 15 snakes. If your target needs more snake impact than 15 snakes can provide, the actual median will fall short of the target.
• Rewards Quantity too high for the board — due to the no-adjacency rule, rewards need spacing. On a 5×5 board, you can fit roughly 6–7 rewards max even if you set quantity to 50. The system will place as many as it can and silently stop.
• Target Rewards Per Run > Rewards Quantity — you can never collect more rewards than exist on the board. The actual median will cap at the quantity available.
• Target Rewards Per Run is high but board has many snakes — snakes can route players past reward tiles, reducing collection rates. The target rewards setting doesn't directly control snake/ladder placement, so heavy-snake boards will naturally have lower reward collection.
• Placement collisions — each snake uses 2 tiles (head + tail), each ladder uses 2 tiles (bottom + top), and each reward uses 1 tile. Tile 1 (start) and the final tile are always reserved. On small boards, there may not be enough free tiles for all requested entities.

Tip: After generating, always check the simulation results to see the actual median vs your target. If they don't match, the board constraints made the exact target impossible — try adjusting your settings or manually editing the board.

Note: Clicking "Apply Settings & Generate Board" will regenerate the entire board (new snakes, ladders, rewards placement) and auto-run a simulation. Use this when you want a fresh board.

Export / Import — save your board layout as a .json file or load a previously saved one.

2 Play the Game

Roll Dice — roll once and move your token.

Auto Roll — rolls automatically every 400ms until you win or stop.

New Game — reset and start over.

TileEffect
🐍Snake — slides you down to a lower tile
🪜Ladder — climbs you up to a higher tile
🎁Reward — collect it (turns to 💎)
🏆Final — reach here to win!
3 Edit the Board

In the right panel, select an edit tool then click tiles:

Snake — click HEAD first, then TAIL (lower number).

Ladder — click BOTTOM first, then TOP (higher number).

Reward — click any tile to toggle.

Remove — click any tile to clear it.

Changes save automatically.

4 Run Simulation

There are two simulation controls with different behaviors:

LocationControlBehavior
Left PanelApply Settings & Generate BoardRegenerates the board with new snakes, ladders & rewards, then auto-runs a simulation. Use when you want a fresh board.
Right PanelRun SimulationRuns a simulation on the current board as-is without changing it. Use after manual edits to see updated stats.

Both share the same sample size setting — changing it in either panel syncs the other.

Results show: roll statistics (median, mean, percentiles), roll distribution histogram, reward collection rates, per-reward table, and board summary.

Export Board — download your board as a .json file.
